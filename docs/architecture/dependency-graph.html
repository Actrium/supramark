supramark — 项目结构与依赖关系

分层架构视图：用户层 → TS 编排 → wasm 桥 → Rust 端口 → 字体测量底座。截至 commit 165645ab (2026-05-11)。

Host Browser API TypeScript 应用 TS 集成胶水 wasm 包 (npm) Rust 端口 crate Rust 底座 crate 运行时调用 cfg 编译期分流 host bridge

关键点说明：

① TS 应用层 (@supramark/core + features + web/rn renderer) 不直接依赖任何 Rust crate；所有图表能力通过 @supramark/diagram-engine 抽象出口。

② 字体测量分两路：实线是常规调用链（编译产物间的依赖）；虚线是 Cargo cfg(target_arch="wasm32") 编译期分流；绿线是 host 浏览器 bridge 注入路径。

③ font-metrics trait 在 plantuml / mermaid / d2 三家保持纯单调原语；d2 因 Go 上游 multi-line + line_height_factor 的 stateful 语义额外引入 D2Metrics sub-trait。

④ wasm bundle 现已不再嵌入 TTF 字体数据：d2-little 从 5.30 MB 缩到 3.63 MB（-1.67 MB / -32%）；plantuml/mermaid 嵌入的 font-metrics Latin 子集 ~970 KB 也仅在 native + SSR 用，wasm 端走 host canvas。

⑤ 待优化项见 GitHub issues：#5（plantuml 10 baseline test backlog）、#6（host-callback webfont async loading 文档化）。
